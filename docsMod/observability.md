# 📘 S2J Docs Linter - SLA・指標・トレース

## 1. 可観測性の仕様

本書は、S2J Docs Linter プラットフォームの可観測性の契約を定義します。

可観測性は、プラットフォームのランタイム状態を可視化し、性能/健全性/信頼性を継続的に評価するための契約です。

対象は、下記のコンポーネントとします。

* Core ランタイム
* REST ランタイム
* SDK
* SDK ジェネレーター
* 「利用側」連携

## 2. 目的

可観測性は、下記を目的とします。

* ランタイム・ヘルスの把握
* パフォーマンス・バジェットの維持
* 失敗解析
* 継続的改善
* 運用可視性

## 3. 可観測性の原則

プラットフォームは、下記の原則に従います。

* デフォルトで観測可能
* 指標 First
* 構造化ログ記録
* 追跡可能な実行
* オーバーヘッドの少ないモニタリング

## 4. パフォーマンス・バジェット

ランタイムは、「パフォーマンス・バジェット」を満たすことを目標とします。

### バジェット・カテゴリー

* 起動時間
* リクエスト・レイテンシー
* 検証スループット
* メモリー使用量
* CPU 使用率

### ルール

パフォーマンス・バジェットは、ランタイム・プロファイルごとに定義できます。

## 5. 契約

### ログ記録契約

ランタイムは、構造化ログを生成します。

### トレーシング契約

リクエストは、トレース可能であることとします。

## 6. レイテンシー契約

### 標準指標

* 平均レイテンシー
* P95 レイテンシー
* P99 レイテンシー

### ルール

レイテンシーは、継続的に収集します。

## 7. スループット契約

### 指標

* 秒間のリクエスト
* 秒間の検証
* 一括スループット

## 8. リソース利用契約

### 指標

* CPU
* メモリー
* ファイル・ハンドル
* スレッド

### ルール

ランタイムは、リソース使用状況を公開できます。

## 9. ログ記録

ランタイムは、構造化ログを生成します。

### ログ・レベル

* Trace
* Debug
* Info
* Warn
* Error

### ルール

Secret および個人情報を、出力してはなりません。

## 10. 指標契約

### 必須指標

* 検証回数
* エラー回数
* キャッシュヒット率
* 起動時間
* アクティブ・ランタイム

### ルール

指標は、機械可読で取得できることとします。

## 11. トレーシング

リクエストは、トレース可能であることとします。

### 標準属性

* トレース ID
* リクエスト ID
* ランタイム
* プロファイル
* バージョン

### ルール

トレース ID は、ランタイムを跨いで維持します。

## 12. ヘルス契約

### 必須の状態

* 起動
* 準備完了
* 正常
* 劣化
* 不健全

### ルール

ヘルスは、機械可読で公開します。

## 13. アラート契約

### アラートの対象

* エラー率
* タイムアウト
* 起動失敗
* リソース枯渇

### ルール

重大アラートは、通知対象とします。

## 14. パフォーマンス回帰の方針

パフォーマンス回帰を検出します。

### 必須テスト

* ベンチマーク
* 回帰テスト
* 負荷テスト

### ルール

パフォーマンス・バジェットを超えた変更は、レビュー対象とします。

## 15. ランタイム・プロファイル指標

### 標準プロファイル

* ブラウザー
* WordPress
* CLI
* REST

### ルール

プロファイルごとに、異なるバジェットを持てます。

## 16. 完了条件

可観測性の仕様は、下記を実装した時点で完成とみなします。

* 可観測性の原則
* パフォーマンス・バジェット
* レイテンシー契約
* スループット契約
* リソース利用契約
* ログ記録契約
* 指標契約
* トレーシング契約
* ヘルス契約
* アラート契約
* パフォーマンス回帰の方針
* ランタイム・プロファイル指標
* ADR (アーキテクチャ決定記録)

## 17. ADR (アーキテクチャ決定記録)

### ADR-OBS-001

#### タイトル

* 指標 First

#### 決定

* 指標を、可観測性の中心とする。

### ADR-OBS-002

#### タイトル

* 構造化ログ記録

#### 決定

* ログは、構造化フォーマットとする。

### ADR-OBS-003

#### タイトル

* パフォーマンス・バジェット

#### 決定

* パフォーマンス・バジェットを、契約化する。

### ADR-OBS-004

#### タイトル

* 分散トレーシング

#### 決定

* トレース ID を、ランタイム間で共有する。

### ADR-OBS-005

#### タイトル

* ヘルス駆動オペレーション

#### 決定

* ヘルス・ステータスを、ランタイムの標準契約とする。
